## Сценарії використання

### Для громадян

**Ранкова перевірка:**
```
Доброго ранку! Які були тривоги вночі в Києві?
```

**Перед виходом:**
```
Чи безпечно зараз виходити на вулицю в Одесі?
```

**Перед сном:**
```
Скільки разів була тривога сьогодні в моєму місті (Харків)?
```

### Для родини

**Перевірка близьких:**
```
Мої батьки в Вінниці. Чи є там зараз тривога?
```

### Для бізнесу

**Логістика:**
```
Покажи статус тривог у Дніпрі, Запоріжжі та Кривому Розі
```

**Планування робіт:**
```
Яка була історія тривог в Одесі за останній тиждень?
```

### Для розробників

**Отримання даних:**
```
Покажи JSON відповідь про активні тривоги
```

**Моніторинг:**
```
Перевір номер останньої дії в системі тривог
```

## Поради по використанню

### Ефективні запити

✅ **Добре:**
- "Чи є тривога в Києві?"
- "Покажи історію тривог для Харківської області"
- "Які області зараз без тривог?"

❌ **Погано:**
- "Тривога" (занадто коротко)
- "Дай інфу" (неконкретно)
- "123" (незрозуміло)

### Комбінування запитів

Ви можете запитувати кілька речей одночасно:

```
Перевір тривоги в Києві, Харкові та Одесі, 
і скажи які області зараз без тривог
```

### Контекстні запити

OpenClaw запам'ятовує контекст:

```
Користувач: Чи є тривога в Києві?
Агент: Так, наразі в Києві активна тривога...

Користувач: А коли вона почалася?
Агент: Тривога в Києві почалася о 14:25...

Користувач: Покажи історію за сьогодні
Агент: Історія тривог для м. Київ за 28.02.2024...
```

## Автоматизація

### Регулярні перевірки

Налаштуйте OpenClaw на регулярні перевірки (через cron або hooks):

```json
{
  "automation": {
    "cron": {
      "check-alerts": {
        "schedule": "*/15 * * * *",
        "command": "Перевір тривоги в Києві та повідом якщо є зміни"
      }
    }
  }
}
```

### Сповіщення

```json
{
  "automation": {
    "webhook": {
      "alert-started": {
        "url": "https://your-webhook.com/alert",
        "trigger": "Коли з'являється нова тривога в Київській області"
      }
    }
  }
}
```

## Питання та відповіді

**Q: Як часто оновлюється інформація?**  
A: API оновлюється в реальному часі. Запитуйте коли потрібно.

**Q: Чи можна отримати дані по окупованих територіях?**  
A: Дані доступні, але можуть бути неточними.

**Q: Скільки можна робити запитів?**  
A: API має ліміти, але для особистого використання їх достатньо.

**Q: Чи зберігається історія запитів?**  
A: OpenClaw може зберігати контекст розмови, але не зберігає історію API запитів.
